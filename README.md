BYC Cloud Platform

|                      |                      |
|----------------------|----------------------|
| JDK                  | 17                   |

| Spring Cloud Alibaba | 2023.0.1.2           |
| Registry/Config      | Nacos                |

| Persistence          | MyBatis-Plus + MySQL |
| Cache                | Redis                |

| OpenAPI              | springdoc 2.x        |
| AI                   | LangChain4j          |

## Layout

```
byc-cloud-platform/
├── pom.xml                         # Aggregator + dependency management
├── gateway/
│   └── gateway-service/            # Spring Cloud Gateway entrypoint
├── services/                       # Business services (added via `sca add-service`)
├── jobs/                           # Scheduled jobs    (added via `sca add-job`)
├── ai/                             # AI services      (added via `sca add-ai-service`)
├── common/
│   ├── common-core/                # Result/exception/utils
│   ├── common-web/                 # Web filters, exception handlers, OpenAPI
│   ├── common-security/            # JWT / auth
│   └── common-redis/               # Redis template + cache helpers
├── deploy/
│   ├── docker/docker-compose.yml   # Local stack (Nacos / MySQL / Redis)
│   ├── k8s/                        # Plain manifests per service
│   └── helm/                       # Helm chart (single chart, services parameterized)
├── scripts/                        # build / start-local / deploy
├── docs/                           # Architecture & runbooks
└── .github/workflows/ci.yml        # GitHub Actions matrix CI
```

## Quick start (local)

```bash
# 1. Spin up the local stack (Nacos, MySQL, Redis)
./scripts/start-local.sh

# 2. Build everything
./scripts/build.sh

# 3. Run any service
cd gateway/gateway-service
../../mvnw spring-boot:run
```

## Adding modules

```bash
sca add-service   user-service
sca add-service   order-service
sca add-job       sync-job
sca add-ai-service rag-service
sca add-common    common-mq
```

Each command:

- creates the module
- updates the root `pom.xml`
- updates `gateway-service/src/main/resources/application.yml` (services & AI services)
- writes `deploy/k8s/<name>.yaml`
- appends the entry to `deploy/helm/values.yaml`
- appends the module to the GitHub Actions matrix
